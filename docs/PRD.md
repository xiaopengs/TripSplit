# 拼途记账（TripSplit）—— 产品需求文档（PRD）

> **版本**：v1.0  
> **日期**：2026-04-05  
> **状态**：开发中  
> **技术栈**：微信原生小程序（非 Taro）

---

## 1. 产品概述

### 1.1 产品定位
拼途记账是一款面向旅行场景的**多人协作记账工具**，核心目标是实现从"零门槛创建"到"AI/手动双轨记账"，最终完成"智能无感结算"的全闭环。

### 1.2 核心价值主张
| 价值点 | 描述 |
|--------|------|
| **零门槛创建** | 3 秒建账本，定位自动选币种，预设影子成员 |
| **AI 拍照识别** | 拍小票即走，后台 AI 自动识别金额与类目 |
| **手动极速录入** | 半屏面板 + 大号键盘，支持简易运算 |
| **智能结算** | 最少转账次数算法，自动对冲，一键请款 |
| **影子成员系统** | 未加入前即可记账，后续扫码认领无缝迁移 |

### 1.3 目标用户
- **主要用户**：结伴旅行的年轻人（18-35岁）
- **使用场景**：国内/出境旅行、朋友聚餐、AA 制活动

### 1.4 品牌信息
- **产品名称**：拼途记账
- **品牌色**：主绿色 `#34C759`（活力、旅行、金钱）
- **辅助色**：橙色 `#FF9500`（Logo 渐变）
- **Logo**：地图定位图标 + 拼图元素，体现"拼途"概念

## 2. 功能模块详细需求

### 2.1 账本创建与管理（The Genesis）

#### 2.1.1 创建入口
- **空状态首页**：页面中心显示巨大的 `+` 新建账本按钮，配合引导文案和 Logo 动画
- **侧边栏入口**：支持多账本切换，区分「进行中」和「已归档」账本

#### 2.1.2 创建表单字段
| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 账本名称 | 文本输入 | 必填 | 例："曼谷大吃大喝之旅"，最多 20 字 |
| 默认币种 | 选择器 | 定位 API 判断 | 境外→当地货币；境内→人民币 CNY |
| 视觉皮肤 | 6 色选择 | 随机 | 低饱和度纯色背景，6 种可选 |
| 开始日期 | 日期选择器 | 今天 | 可修改 |
| 成员列表 | 动态列表 | 创建者本人 | 支持添加影子成员 |

#### 2.1.3 影子成员系统（Shadow System）
- **定义**：管理员在创建账本时预先添加的成员，尚未绑定真实微信用户
- **数据结构**：
  ```
  {
    id: "shadow_xxx",
    name: "大壮",
    avatar: "",
    is_claimed: false,
    claimed_by: null,
    created_at: timestamp
  }
  ```
- **认领流程**：
  1. 管理员生成分享链接/二维码
  2. 真实用户通过微信打开链接
  3. 弹出身份认领弹窗：「你是 [大壮] 还是 [二狗]？」
  4. 用户选择后，`is_claimed = true`，历史记录归属权迁移至该用户 UID
- **影子成员显示**：成员列表中用黄色「影子」标签标识，待认领状态显示「待认领」文字

### 2.2 交互核心：扇形弹出菜单（The FAB Fork）

#### 2.2.1 按钮规格
- **位置**：右下角固定悬浮，距底部 80px，距右侧 24px
- **尺寸**：56px 圆形按钮
- **颜色**：品牌绿 `#34C759`
- **图标**：白色 `+` 号

#### 2.2.2 扇形展开交互
```
         ┌───┐
         │手动│ ← 左上偏移 (dx:-60, dy:-40)
    ┌────┴───┴────┐
    │      +      │ ← 主按钮，旋转 45°
    └────┬───┬────┘
         │拍照│ ← 右上偏移 (dx:+60, dy:-40)
         └───┘
```

- **触发**：点击主按钮展开/收起
- **动画参数**：200ms，Spring 曲线 `cubic-bezier(0.175, 0.885, 0.32, 1.275)`
- **触觉反馈**：`wx.vibrateShort({ type: 'light' })`
- **背景遮罩**：半透明黑色遮罩，点击收起

#### 2.2.3 子按钮功能
| 按钮 | 图标 | 行为 |
|------|------|------|
| 手动记一笔 | ✏️（编辑） | 呼起半屏录入面板 |
| 拍照/OCR | 📷（相机） | 呼起全屏相机拍照 |

### 2.3 记账路径 A：AI 拍照识别（Snap & Scan）

#### 2.3.1 极速捕获流程
```
[点击拍照] → [调用 wx.chooseMedia] → [立即存入本地 + 显示 loading]
            → [后台上传图片至 OSS] → [调用 AI OCR 接口]
            → [异步更新账单详情] → [推送至待整理收件箱]
```
**关键原则**：快门按下即入库，不阻塞用户操作。

#### 2.3.2 AI 类目映射
| 类目 | 图标 | 关键词示例 |
|------|------|-----------|
| 餐饮 | 叉勺 | 餐厅、饭馆、火锅、咖啡、奶茶、小吃 |
| 交通 | 汽车 | 出租车、地铁、机票、高铁、加油、停车 |
| 住宿 | 床铺 | 酒店、民宿、公寓、客栈 |
| 门票 | 票券 | 景点、博物馆、演出、电影 |
| 购物 | 袋子 | 商场、超市、便利店、纪念品 |
| 其他 | 方块 | 不匹配以上类目时默认 |

#### 2.3.3 待整理收件箱（Inbox）
- **入口**：首页 Tab 栏中间位置「待整理」，右上角红点数字提示未确认数量
- **卡片内容**：收据缩略图 + AI 识别金额 + 自动选定类目标签
- **手势操作**：右滑确认入账（绿色勾选），左滑删除废片（红色垃圾桶）
- **空状态**：「待整理列表为空」+ 引导文案

#### 2.3.4 OCR 异常处理
| 异常场景 | 处理方式 |
|----------|---------|
| 金额置信度 < 0.6 | 金额栏显示红色 "?"，点击呼起键盘纠错 |
| 无法识别金额 | 标记为"待补录"，引导用户手动输入 |
| 无小票/非票据图像 | AI 识别图像内容描述作为备注 |
| 图片模糊/遮挡 | 提示"图片不清晰，请重拍或手动记账" |

### 2.4 记账路径 B：手动记一笔（Manual Zen）

#### 2.4.1 面板规格
- **类型**：从底部滑出的半屏模态窗口
- **高度**：屏幕高度的 70%
- **遮罩**：点击上半部分遮罩可关闭
- **标题栏**：「记一笔」+ 关闭按钮 ×

#### 2.4.2 金额键盘组件
- **布局**：3×4 网格（7-8-9 / 4-5-6 / 1-2-3 / .-0-⌫）
- **字体**：36px 粗体等宽字体
- **运算支持**：基础加减运算
- **最大金额**：999,999.99

#### 2.4.3 类目磁贴
- **布局**：横向滚动的 6 个大图标磁贴
- **选中效果**：呼吸灯动画（scale 1.05 + 边框发光）
- **类目**：餐饮/交通/住宿/门票/购物/其他

#### 2.4.4 智能分摊
- **默认策略**：全员均分
- **UI**：成员头像列表，默认全选，点击排除
- **支持自定义每人金额**

#### 2.4.5 扩展字段
- **备注**：文本输入 + 语音转文字
- **图片**：相册选图补充（最多 3 张）

### 2.5 结算中心（Smart Settlement）

#### 2.5.1 最少转账算法
采用贪心优化算法：
1. 计算每个成员净收支
2. 正负分别排序
3. 贪心匹配最大应付款 vs 最大应收款
4. 重复至余额归零
5. 自动对冲中间环节

#### 2.5.2 结算方案 UI
- 已优化为最少转账次数（N 笔）
- 每张卡片：付款人→收款人→金额
- 操作：「标记已付」+ 「下顿他请」
- 抹零开关：四舍五入到元

## 3. 全链路用户场景

### 场景一：创建账本
登录 → 新建账本 → 输入名称 → 定位选币种 → 选肤色 → 添加影子成员 → 完成

### 场景二：AI 拍照记账
点(+) → 拍照 → 锁屏走人 → 后台 AI 识别 → Inbox 红点提醒 → 右滑确认入账

### 场景三：成员认领
管理员分享二维码 → 好友扫码 → 选择身份认领 → 绑定成功 → 看到账单

### 场景四：手动记账
点(+) → 手动记一笔 → 输金额 → 选类目 → 确认分摊 → 提交

### 场景五：结算
点结算 Tab → 系统计算最优方案 → 生成转账建议 → 请款卡分享

## 4. 非功能性需求

### 4.1 数据幂等性
- 请求携带唯一 request_id（UUID）
- 后端幂等校验
- 本地 pending 状态防重复提交

### 4.2 多币种汇率
- 每日定时拉取汇率
- 记账时固化汇率快照
- 结算可按最新汇率换算

### 4.3 本地缓存
| 数据类型 | 缓存方式 | 同步策略 |
|----------|---------|---------|
| 当前账本 | wx.setStorage | 实时双向同步 |
| 流水列表 | wx.setStorage | 增量 diff 同步 |
| 待整理 Inbox | wx.setStorage | 仅本地，确认后上传 |
| 离线队列 | wx.setStorage | 网络恢复 FIFO 上传 |

### 4.4 计算精度
- 所有金额以分为单位存储（Integer）
- 后端使用 BigDecimal
- 禁止浮点数运算
- 余数分配给支付者

### 4.5 性能要求
- 首屏 < 1.5s
- FAB 动画 ≤ 200ms
- AI OCR < 5s（后台异步）
- 超 50 条流水启用虚拟滚动
